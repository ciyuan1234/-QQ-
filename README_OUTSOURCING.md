# 农场花园种花偷花项目 - 外包交付标准版

此版本已针对现代服务器环境进行重构和优化，确保“拿来即用”并符合外包交付标准。

## 🛠 已完成的重构项

1. **后端数据库底层重构**：
   - 解决了原代码在 PHP 7.0+ 环境下因为 `mysql_*` 函数被移除导致无法运行的问题。
   - 在 `php-api/mysql.php` 中实现了 **MySQLi 兼容层**，无需修改成百上千处业务代码即可在 PHP 7.4/8.x 上完美运行。
   - 统一了数据库连接管理，增加了全局连接对象，提升了性能和稳定性。

2. **前端配置优化**：
   - 优化了 `common/server.js`，预留了清晰的配置项占位符和注释。
   - 增加了防御性编程（如 `get_res` 的空值判断），减少由于配置不当导致的白屏或崩溃。

3. **安全性与逻辑加固 (Security & Logic Audit)**：
   - **防 SQL 注入**：对所有核心业务接口（采摘、售卖、播种等）进行了输入清理，通过 `mysql_real_escape_string` 过滤用户敏感参数，从根源杜绝了数据库被恶意操控的风险。
   - **身份权限校验**：完善了基于 PHP Session 的 OpenID 校验机制，确保用户只能操作自己的资产，无法跨权越界。
   - **并发采摘保护**：优化了采摘逻辑的原子性，增加了库存水位检查，防止多人并发操作导致的逻辑坏账。

## 🚀 部署指南

### 第一步：后端部署 (PHP)
1. 将 `▓┐╩≡╦∙╨Φ╬─╝■/php-api` 目录上传到服务器（建议重命名为 `php-api`）。
2. 将 `▓┐╩≡╦∙╨Φ╬─╝■/cdn-images` 目录上传到服务器或 CDN。
3. **数据库配置**：
   - 导入 `▓┐╩≡╦∙╨Φ╬─╝■/db.sql` 到 MySQL 数据库（建议数据库名设为 `wpoy`）。
   - 依次导入 `amazpot_seeds.sql` 和 `amazpot_shop.sql` 初始数据。
   - 修改 `php-api/mysql.php` 中的数据库连接参数（地址、用户名、密码）。
   - 在 `php-api/mysql.php` 中填写你的微信小程序 `AppID` 和 `AppSecret`。

### 第二步：前端配置 (uni-app)
1. 使用 HBuilderX 打开本项目。
2. 修改 `common/server.js` 中的 `HOST` 为你的域名。
3. 修改 `manifest.json` 中的微信小程序 `appid`。
4. 运行 -> 运行到小程序模拟器 -> 微信开发者工具。

## ⚠️ 交付须知 (Logic Notes)
- **微信授权**：由于微信官方 API 调整，老版的 `getUserInfo` 可能无法获取真实的头像和昵称。建议在上线前参考微信最新文档，在个人中心增加一个“修改头像昵称”的功能。
- **PHP 版本**：推荐使用 PHP 7.4，兼容性最佳。
- **文件权限**：确保 `php-api/access_token.json` 具有写权限（建议设为 777），否则微信接口调用会失败。

## 📱 微信小程序适配说明 (WeChat Specific)

本版本已针对微信小程序环境进行了专项适配，但由于微信官方政策调整，以下两点需要开发者根据实际情况调整：

1. **消息通知升级**：
   - 项目中使用的 `templateMessage`（模板消息）已被微信官方废弃。
   - **现状**：代码中保留了相关逻辑以供参考，但实际下发通知会失效。
   - **适配方案**：需在微信后台申请“订阅消息”，并将后端 `sqlutils.php` 中的 `sendNotice` 接口更新为 `subscribeMessage.send`。

2. **头像昵称规范**：
   - 旧版的 `getUserInfo` 按钮在当前微信环境中不再弹出授权框，只会返回“微信用户”。
   - **适配方案**：建议在游戏个人设置页面，使用微信最新的 `button open-type="chooseAvatar"` 和 `input type="nickname"` 能力让用户手动设置头像和昵称。

3. **隐私协议**：
   - 需在微信小程序后台配置并启用《用户隐私保护指引》。
   - `manifest.json` 中已默认开启 `__usePrivacyCheck__`。
